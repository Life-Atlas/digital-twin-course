How to Connect LinkedIn
to Claude Code via MCP

Post, message, and search LinkedIn directly from your terminal. One-time OAuth setup, 60-day auto-refresh, zero lock-in. The complete guide we wish existed when we started.

What You're Building

Claude Code  -->  LinkedIn MCP Server  -->  LinkedIn API
   (you)         (runs locally)            (posts, messages, profiles)

The MCP server runs on your machine. It talks to LinkedIn's official API using OAuth 2.0. You log in once, get a token that lasts 60 days, and Claude Code can use 25+ LinkedIn tools without you touching a browser again.

1

Create a LinkedIn Developer App

  1. 1. Go to linkedin.com/developers
  2. 2. Click Create App
  3. 3. Fill in:
    • App name: Whatever you want (e.g., "MyApp MCP")
    • LinkedIn Page: Select your company page (required)
    • Logo: Upload any image
  4. 4. Click Create App

You'll land on the app's Settings page. Note your Client ID at the top.

2

Add Required Products

In the Developer Portal, click the Products tab. Request access to these three (they auto-approve):

ProductWhy
Share on LinkedInCreate posts, comments, reactions
Sign In with LinkedIn using OpenID ConnectRequired for OAuth login
Verified on LinkedInProfile verification access

Click Request Access for each. They activate immediately.

3

Configure the Redirect URI

  1. 1. Click the Auth tab
  2. 2. Under Authorized redirect URLs, click the pencil icon
  3. 3. Add exactly this URL:
http://localhost:44002/auth/linkedin/callback

This is http, not https — that's correct. It's localhost, so no encryption needed.

4

Copy Your Client Secret

Still on the Auth tab:

  1. 1. Find Primary Client Secret
  2. 2. Click the eye icon to reveal it
  3. 3. Copy it — you'll need it in the next steps

It looks like: WPL_AP1.abcDEF123456.xyzABC==

5

Add the MCP Server to Claude Code

Open a terminal and run:

claude mcp add linkedin -s local \
  -e LINKEDIN_CLIENT_ID=YOUR_CLIENT_ID \
  -e "LINKEDIN_CLIENT_SECRET=YOUR_CLIENT_SECRET" \
  -- npx -y linkedin-mcp-server

Replace YOUR_CLIENT_ID and YOUR_CLIENT_SECRET with your actual values.

Wrap the secret in double quotes because it contains = signs.

6

Authenticate (One-Time Browser Login)

The MCP server tries to open your browser automatically, but on Windows it often fails silently. Here's the reliable method:

6a. Start the callback server

Save this as linkedin-auth.js and run with node linkedin-auth.js:

const http = require('http');
const https = require('https');
const fs = require('fs');
const path = require('path');

// REPLACE THESE WITH YOUR VALUES
const CLIENT_ID = 'YOUR_CLIENT_ID';
const CLIENT_SECRET = 'YOUR_CLIENT_SECRET';
const REDIRECT_URI = 'http://localhost:44002/auth/linkedin/callback';

const server = http.createServer((req, res) => {
  const u = new URL(req.url, 'http://localhost:44002');
  if (u.pathname === '/auth/linkedin/callback' && u.searchParams.get('code')) {
    const code = u.searchParams.get('code');
    const postData =
      'grant_type=authorization_code&code=' + code +
      '&redirect_uri=' + encodeURIComponent(REDIRECT_URI) +
      '&client_id=' + CLIENT_ID +
      '&client_secret=' + encodeURIComponent(CLIENT_SECRET);

    const tokenReq = https.request({
      hostname: 'www.linkedin.com',
      path: '/oauth/v2/accessToken',
      method: 'POST',
      headers: { 'Content-Type': 'application/x-www-form-urlencoded' }
    }, (tokenRes) => {
      let data = '';
      tokenRes.on('data', c => data += c);
      tokenRes.on('end', () => {
        const tokens = JSON.parse(data);
        if (tokens.error) {
          console.log('ERROR:', tokens.error_description);
          res.writeHead(400, { 'Content-Type': 'text/html' });
          res.end('<h1>Error: ' + tokens.error_description + '</h1>');
          server.close();
          process.exit(1);
          return;
        }
        const tokenDir = path.join(
          process.env.USERPROFILE || process.env.HOME,
          '.linkedin-mcp'
        );
        if (!fs.existsSync(tokenDir))
          fs.mkdirSync(tokenDir, { recursive: true });
        fs.writeFileSync(
          path.join(tokenDir, 'tokens_default.json'),
          JSON.stringify({
            access_token: tokens.access_token,
            expires_in: tokens.expires_in,
            refresh_token: tokens.refresh_token || null,
            scope: tokens.scope,
            token_type: tokens.token_type,
            created_at: Date.now()
          }, null, 2)
        );
        res.writeHead(200, { 'Content-Type': 'text/html' });
        res.end('<h1>LinkedIn authenticated! Close this tab.</h1>');
        console.log('SUCCESS! Token saved. Expires in ' +
          Math.round(tokens.expires_in / 86400) + ' days.');
        server.close();
        process.exit(0);
      });
    });
    tokenReq.write(postData);
    tokenReq.end();
  }
});

server.listen(44002, () => {
  console.log('Callback server ready on port 44002.');
  console.log('Now open the auth URL in your browser (see 6b).');
});

// Auto-close after 5 minutes
setTimeout(() => { server.close(); process.exit(1); }, 300000);

6b. Open the auth URL in your browser

While the callback server is running, construct this URL (replace YOUR_CLIENT_ID):

https://www.linkedin.com/oauth/v2/authorization?response_type=code&client_id=YOUR_CLIENT_ID&redirect_uri=http%3A%2F%2Flocalhost%3A44002%2Fauth%2Flinkedin%2Fcallback&scope=openid%20profile%20email%20w_member_social&state=auth123

Paste the URL into your browser. LinkedIn will ask you to log in and authorize the app. Click Allow. Your terminal should print SUCCESS.

6c. Verify the token

cat ~/.linkedin-mcp/tokens_default.json

You should see a JSON file with access_token, expires_in (5183999 = 60 days), and other fields.

7

Restart Claude Code & Test

Close and reopen Claude Code. The LinkedIn MCP server will start with your credentials and find the saved token.

Try these commands:

Get my LinkedIn profile
Post on LinkedIn: "Testing my new AI workflow"

If it works, you're done. All 25+ LinkedIn tools are now available in Claude Code.

Troubleshooting

ProblemSolution
"Client authentication failed" Wrong client secret. Reveal it again on the Auth tab and copy carefully. Watch for = signs at the end.
"Bummer, something went wrong" URL got truncated. Copy the full URL to clipboard instead of clicking a link.
"redirect_uri parameter" error The redirect URI must exactly match http://localhost:44002/auth/linkedin/callback
Browser doesn't open Normal on Windows when running as MCP subprocess. Use the manual method (Step 6).
Timeout after 5 minutes The callback server didn't receive the redirect. Check that port 44002 isn't blocked by firewall.
Token expired (60 days) Re-run Step 6. Same process, takes 2 minutes.

What NOT to Do

  • ✗ Don't use https for the redirect URI — it's localhost, http is correct
  • ✗ Don't use paid LinkedIn MCP services (LiGo, Composio) — this free setup does the same thing
  • ✗ Don't use cookie-based scrapers — they can't post and risk account bans
  • ✗ Don't generate a new client secret unless the old one is compromised — it invalidates the old one

Architecture

You type in Claude Code
        |
        v
Claude calls mcp__linkedin__create_post
        |
        v
LinkedIn MCP Server (npx linkedin-mcp-server)
  - Reads token from ~/.linkedin-mcp/tokens_default.json
  - Calls LinkedIn REST API with Bearer token
  - Returns result to Claude
        |
        v
Claude shows you the result

The token lives locally on your machine. No cloud. No third-party servers. No lock-in. The MCP server is an npm package you can replace anytime.
